# StoryForge - 창작자를 위한 웹 플랫폼 기획 초안

> 작성일: 2026-03-19
> 버전: v0.1 (초안)
> 상태: 기획 중

---

## 1. 프로젝트 개요

### 비전
일반 사용자들이 로그인하여 각자의 창작활동(소설 집필 중심)을 할 수 있도록
**글 작성과 자료 정리를 도와주는 웹 플랫폼**

### 프로젝트명 (가칭)
**StoryForge** - 이야기를 만들어가는 공간

### 핵심 가치
- **집중**: 글쓰기에 집중할 수 있는 깔끔한 에디터
- **정리**: 캐릭터, 세계관, 플롯 등 창작 자료를 체계적으로 관리
- **성장**: 작품을 공유하고 피드백을 주고받는 커뮤니티

---

## 2. 대상 사용자

| 유형 | 설명 |
|------|------|
| 웹소설 작가 | 연재 소설을 집필하는 아마추어/프로 작가 |
| 취미 작가 | 단편, 에세이 등을 쓰는 일반 사용자 |
| 창작 동아리 | 함께 작품을 만들고 피드백하는 그룹 |

---

## 3. 핵심 기능 (MVP)

### Phase 1 - 기본 플랫폼 (현재 목표)
- [ ] **회원가입/로그인**: 이메일 기반 인증
- [ ] **대시보드**: 내 작품 목록, 최근 활동, 통계
- [ ] **글쓰기 에디터**: 마크다운/리치텍스트 지원 에디터
- [ ] **작품 관리**: 작품 생성, 챕터 구성, 저장/불러오기
- [ ] **자료실**:
  - 캐릭터 프로필 (기획 디테일 작업 중)
  - 세계관 설정: **폴더 단위 노트 관리**, 이름순/생성일순/수정일순 정렬 지원
  - 플롯 메모

### Phase 2 - 커뮤니티
- [ ] 작품 공개/비공개 설정
- [ ] 다른 사용자 작품 열람
- [ ] 댓글 및 피드백 시스템
- [ ] 작가 프로필 페이지

### Phase 3 - 고급 기능
- [ ] AI 글쓰기 보조 (맞춤법, 문체 제안)
- [ ] 공동 집필 (실시간 협업)
- [ ] 작품 내보내기 (PDF, EPUB)
- [ ] 버전 히스토리

## 4. 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| Frontend | React 19 + TypeScript | 기존 프로젝트 기반 |
| Build | Vite 8 | 빠른 개발 환경 |
| Routing | React Router v7 | SPA 라우팅 |
| Styling | CSS Modules + 커스텀 CSS | 가볍고 유연함 |
| State | React Context + useReducer | MVP에 충분 |
| Backend | (Phase 2) Firebase or Supabase | 빠른 백엔드 구축 |
| Deploy | GitHub Pages | 무료, 간편한 정적 배포 |

## 5. 페이지 구조 (사이트맵)

```
/ (랜딩 페이지)
├── /login (로그인)
├── /signup (회원가입)
├── /dashboard (대시보드 - 로그인 후)
│   ├── /works (내 작품 목록)
│   │   └── /works/:id (작품 상세)
│   │       ├── 집필 탭 (에디터 + 챕터 사이드바)
│   │       ├── 챕터 탭 (전체 챕터 관리)
│   │       ├── 캐릭터 탭 (프로필 카드)
│   │       └── 세계관 탭 (폴더 기반 노트 + 정렬)
│   └── /profile (내 프로필 + 설정)
└── /explore (작품 탐색 - Phase 2)
```

## 6. 화면 설계 요약

### 랜딩 페이지 (/)
- 히어로 섹션: 서비스 소개 + CTA 버튼
- 주요 기능 소개 (3개 카드)
- 샘플 에디터 미리보기
- 시작하기 버튼

### 대시보드 (/dashboard)
- 사이드바 네비게이션
- 내 작품 카드 그리드
- 빠른 시작 (새 작품 만들기)
- 최근 편집 이력

### 글쓰기 에디터 (/works/:id/write)
- 좌: 챕터 목록 사이드바
- 중앙: 에디터 (집중 모드 지원)
- 우: 자료 패널 (토글)

## 8. 당장의 목표 (2026-03-21~)

> 프론트엔드 기획/화면 보강 + 백엔드/인프라 기반 구축을 **병행**

### A. 프론트엔드 — 기획 보강 & 미완성 화면

#### A-1. 미작동 기능 연결
- [ ] **회원가입 (Signup.tsx)**: 폼 유효성 검사 (이메일 형식, 비밀번호 강도, 중복 체크)
- [ ] **새 작품 생성 (NewWork.tsx)**: 생성한 작품이 실제 목록에 반영되도록 상태 연결
- [ ] **프로필 설정 (Profile.tsx)**: 닉네임/소개 저장 연결, 테마 셀렉터 "준비 중" 해제

#### A-2. 집필 지원 기능 (WorkDetail — 현재 "준비 중")
- [ ] **집중 모드**: 사이드바/헤더 숨기고 에디터만 표시
- [ ] **글자 수 목표**: 목표 설정 + 진행률 바
- [ ] **맞춤법 검사**: 외부 API 연동 또는 자체 사전 기반 (Phase 3 이후 판단)

#### A-3. 화면/UX 개선
- [ ] **타임라인 드래그앤드롭**: 현재 ▲/▼ 버튼만 있음 → DnD 지원
- [ ] **데이터 내보내기/가져오기**: JSON 파일로 작품 백업/복원
- [ ] **모바일 QA**: 타임라인, 관계도 등 소형 화면 대응 점검
- [ ] **에러 바운더리**: 전역 에러 처리 UI

### B. 백엔드 — 서버 & API 구축

#### B-1. 인증 시스템
- [ ] 이메일 기반 회원가입/로그인 API
- [ ] JWT 토큰 발급 + 리프레시 토큰
- [ ] OAuth 소셜 로그인 (Google/카카오 — 선택)
- [ ] 비밀번호 재설정 플로우

#### B-2. CRUD API
- [ ] 작품 (Work) CRUD
- [ ] 챕터/에피소드 (Chapter/Episode) CRUD
- [ ] 캐릭터 + 관계 (Character/Relation) CRUD
- [ ] 세계관 폴더/노트 (WorldFolder/WorldNote) CRUD
- [ ] 타임라인 이벤트 (TimelineEvent) CRUD
- [ ] 용어집 (Glossary) CRUD

#### B-3. 파일 처리
- [ ] 이미지 업로드 API (커버, 캐릭터 이미지)
- [ ] 스토리지 서비스 연결

### C. 인프라 — 서비스 환경 구성

#### C-1. 데이터베이스
- [ ] **Neon (PostgreSQL)** 무료 티어 셋업
- [ ] 스키마 설계 (users, works, chapters, episodes, characters, relations 등)
- [ ] ORM 선정 및 연결 (Prisma 또는 Drizzle)

#### C-2. 호스팅 & 배포
- [ ] **프론트엔드**: GitHub Pages → Vercel 이전 (자동 배포, 프리뷰)
- [ ] **백엔드**: Vercel Serverless Functions 또는 Railway/Fly.io
- [ ] 커스텀 도메인 연결 (선택)
- [ ] HTTPS + CDN 자동 적용

#### C-3. 캐시/세션 (후순위)
- [ ] **Upstash (Redis)**: 사용자 늘어난 후 세션/캐시/Rate Limiting 용도로 도입
